# 见微 PicMark 隐私说明

见微 PicMark 是本地优先的图片查看与标注工具。核心功能在本机完成，不需要账号，也不会上传用户正在处理的图片。

## 默认不会上传的内容

PicMark 不会上传：

- 图片内容
- 文件名
- 文件路径
- `.picmark` 项目内容
- 标注内容
- Windows 用户名
- 电脑名

## 自动检查更新

PicMark 可在启动后延迟检查更新。更新检查是轻量后台请求，最多每 3 天自动检查一次。

更新检查只读取公开的更新清单：

```text
https://raw.githubusercontent.com/Tsang12140/picmark/main/docs/update.json
```

如果网络不可用或请求失败，软件会静默跳过，不影响使用。用户可以在“更新与隐私”里关闭自动检查更新。

## 匿名使用统计

匿名使用统计默认不会强制开启。软件会询问用户是否允许发送匿名统计，用户也可以在“更新与隐私”里随时关闭。

如果用户允许，并且更新清单配置了统计接收地址，PicMark 每天最多发送一次匿名启动统计。统计内容仅用于了解版本使用情况、系统兼容性和低分辨率屏幕占比。

可能发送的字段包括：

```json
{
  "event": "app_start",
  "appVersion": "0.2.3",
  "channel": "setup",
  "os": "Windows 10+",
  "screenBucket": "1400x800",
  "isLowResolution": false,
  "installId": "random-anonymous-id",
  "dateUtc": "2026-06-22"
}
```

说明：

- `installId` 是本机随机生成的匿名 ID，不来自系统账号或硬件序列号。
- `screenBucket` 是分辨率区间，用于判断低分辨率适配效果。
- 原始图片、文件名、文件路径和编辑内容不会被发送。
- 统计失败时静默跳过，不影响软件使用。

## 在线率如何计算

PicMark 不做实时在线监控，不会长期保持后台连接。

如果用户允许匿名统计，并且服务端已配置，可通过匿名启动统计估算：

- 日活跃设备数 DAU
- 7 日活跃设备数 WAU
- 30 日活跃设备数 MAU
- 版本分布
- Windows 版本分布
- 低分辨率设备占比

所谓“在线率”建议按匿名活跃设备估算，例如：

```text
最近 7 天活跃设备数 / 历史匿名安装设备数
```

该数据只代表允许匿名统计的用户，不代表所有用户。
