# 见微 PicMark

> 开源、本地优先的轻量图片查看与标注工具。

PicMark 面向的是很具体的日常场景：打开一张本地图片，持续标注、修改、保存和回看。它更接近一个带编辑能力的本地图片查看器，而不是截图工具、在线设计软件或大型修图软件。

它适合用来标注车位、证件、票据、流程图、截图存档、资料说明图等本地图片。图片默认不上传、不替换原图，只有在用户明确保存、另存为或覆盖时才写入成品。

## 为什么做 PicMark

很多标注需求并不是“截一张图，马上发出去”，而是围绕一张已经存在的本地图片反复修改。

截图工具在这种场景里容易卡住工作流：截图状态会占住屏幕，误退出后标注可能丢失，多次截图还会损失画质。大型设计软件又太重，按钮太多，学习成本高。PicMark 的目标是保留本地图片查看器的直接和轻巧，同时补上常用标注能力。

## 功能亮点

- **本地图片查看**：直接打开 JPG、PNG、BMP、WebP 等常见格式。
- **轻量标注**：框选、圈选、箭头、文字、画笔、马赛克、裁剪。
- **水印工具**：支持文字平铺、证件网格、单个文字、图片 Logo，并支持批量水印。
- **智能手绘美化**：手绘圆、矩形、箭头等图形后可自动美化成更规整的形状。
- **项目文件**：可保存为 `.picmark`，之后继续编辑，不必一次性做完。
- **格式转换**：支持常见图片格式之间的转换，并保留简洁的单图工作流。
- **查看器体验**：查看模式保持浅色、轻量；进入编辑后切换为深色工作台。
- **离线优先**：不需要账号，不上传图片，适合处理本地资料和隐私图片。

## 相比常见工具

PicMark 不追求“功能最多”，而是追求“够用、直观、轻”。

- 相比复杂修图软件，它保留最常用的标注与导出能力，界面更短路径。
- 相比在线设计工具，它不依赖网页、不要求上传图片，也不需要会员。
- 相比普通图片查看器，它不只是查看，还能直接标注、裁切、水印和保存项目。
- 相比截图工具，它更适合反复编辑同一张本地图片，不会因为截图流程中断而丢失进度。

## 设计原则

- **保留原图**：默认不破坏原文件。
- **少即是快**：常用操作优先，不把低频功能堆满界面。
- **低配置友好**：面向普通办公电脑，避免重型依赖和昂贵的后台任务。
- **离线可用**：核心功能不依赖网络服务。

## 安装与使用

### 免安装版

从 Releases 下载免安装包后，解压并运行：

```text
PicMark.exe
```

如果需要让 Windows 的“打开方式”里出现 PicMark，可以在免安装目录中运行注册脚本，或使用安装版。

### 从源码构建

要求：

- Windows 7 SP1 或更高版本
- .NET Framework 4.7.2
- Visual Studio 2022 / MSBuild

构建：

```powershell
.\build-portable.ps1
```

构建完成后，免安装版会输出到：

```text
dist\PicMark-portable
```
